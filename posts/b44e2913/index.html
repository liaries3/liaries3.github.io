Hexo+Github搭建个人网站（超详细）

2021-06-08- 2021-06-12
5.3k- 20m-
网站相关 > 搭建教程-Hexo网站搭建

写给想要拥有一个自己的网站但没有资金成本的小白与曾经的自己。

前言

  虽说是网站，对于个人来说，或许一般叫作博客。

  但我不希望它仅仅是一个博客，而是一个现实生活之外、自由、不用畏惧他人眼光甚至可以独断专行的地方。

  在进入正式搭建之前，请先把下面这段话阅读完毕，清楚在这过程中将会面临什么，再行决定是否要继续搭建。本文以windows系统为例。

博客通常分为动态博客和静态博客：

  本方法所搭建的Hexo为静态博客，所谓静态博客，是指网站无后端，一切内容的修改都是需要直接修改相关代码或配置文件方可生效，包括写文章也需要用markdown语法撰写。

  优点：无需服务器、无需担心网站任何安全问题、速度快。

  缺点：需要修改代码、需要较强的动手能力。

  动态博客：以QQ空间为例，QQ空间中的装扮，发说说只需要点一点，打打字，保存即可完成，类似QQ空间这种操作简单一气呵成的称之为动态博客。

  优点：搭建简单，操作简单，无需markdown语法，点点就行（只有比较高级的DIY才需要修改代码）。

  缺点：服务器需要钱、安全问题需要担心、需要维护各项与服务器有关的东西。

如需了解更多，可自行百度。

准备工作

win10显示文件扩展名（必须操作）

Hexo官网

vscode：用来修改代码，小白建议下载，否则不好控制缩进，针对软件下载慢、英文改中文的方法放在扩展内容部分。

一、如何打开终端（或右击鼠标，再点Git Bash Here，需安装Git）

二、安装必需软件

  • Node.js，建议下载长期支持版（LTS），安装全程点下一步就完事儿了！
  • Git，下载，安装就一直点下一步！（如果官网下载比较慢，可以试试这里）

Node.js 安装成功时也默认安装了npm，在此后将会用到。

如何判断是否安装成功以上软件，打开终端，输入node --version，提示版本号则Node.js安装成功，输入git version，提示版本号则Git安装成功。

三、注册Github账号，如不会注册自行百度。

安装Hexo

  1. 任意位置新建文件夹，并重命名任何你喜欢的名字（英文），从此它就是你的博客根目录。
  2. 在博客根目录右键鼠标，点击Git Bash Here（此操作简称打开终端），输入以下内容，回车。（注：#后面内容为解释，请勿复制）
npm config set registry https://registry.npm.taobao.org
# 将npm源替换为阿里的镜像。之后的安装就会迅速很多了。
npm install hexo-cli -g
# 安装hexo命令集
hexo init
# 初始化hexo

有的人可能会在hexo init这里报错spawn failed、time out、10054这三个类型的错误（因为我只遇到过这三个），按照我的经验是网络问题，多试几次就可以了，如果试了五次还不行就等一段时间再试，成功的标志是文件夹里面多出很多文件。

  1. 完成hexo init这一步标志着hexo的搭建基本完成，博客根目录应包含以下内容：
文件 解释
_config.yml 网站的配置信息，可以在此配置大部分的参数
package.json 已安装的各种插件信息
scaffolds 文章模板
source 资源管理，如各个页面、文章等存放
themes 主题文件夹
  1. 接下来需要进行本地预览，查看是否一切正常，hexo的常用命令如下：
hexo clean
#清空缓存

hexo generate
hexo g #简写
#重新编译

hexo server
hexo s #简写
#打开本地访问
  1. 在博客根目录右键鼠标，点击Git Bash Here，按照上面命令逐一执行，接着打开任意浏览器，在网址栏输入localhost:4000，如果能看到内容即说明搭建成功，本地搭建算是告一段落。

毕竟网站搭建好之后不是给自己在本地看的对吧！而且那个样式还不好看！本地搭建完成还需要将其部署到云端，这样别人才能通过网址访问到，这些将是接下来的内容！

建立Github仓库

注册好Github账号再往下看！

  1. 登陆Github，点击右上角头像左边的+
  2. 点击New repository
  3. 之后你会看到Repository name，这里必须按照下述方法填写，即用户名.github.io。

用户名则是Owner对应下面的那个，是你注册的时候起的，譬如我的用户名是lili，对应的Repository name则是lili.github.io，这个将是日后访问的网址。

  1. 往下 Description (optional)是描述，可填可不填。
  2. 往下Public和Private，必须选择Public，否则无法访问网址。
  3. 直接点击Create repository，完成创建。

更换主题

默认主题并不是很好看，所以我们需要一个好看的主题将它换掉。

如何寻找主题

  1. Easyhexo，里面有一些主题及相关文档,可以自行挑选（小白可以先看看这个）。
  2. Github里面搜索hexo-theme-，可以搜索到大量主题，如果小白不是很会用Github可以暂且先在第一个网站里面找主题试试。
  3. 推荐主题：
    1. Yun，使用文档→文档地址（本站使用）
    2. butterfly，使用文档→文档地址（魔改教程多）
    3. fluid，使用文档→文档地址
    4. shoka，使用文档→[文档地址]{https://shoka.lostyu.me/computer-science/note/theme-shoka-doc/}（不魔改也很精美）

  排名不分先后，都是我认为好看的。

如何安装主题

关于下载：

  绝大部分的使用文档都会告诉你怎么下载，无非是在终端通过命令下载或者在对应Github的release中下载，最后要达到的效果是博客根目录→themes里面要有一个主题文件夹，如果是在Github下载的则直接将主题放置在博客根目录的themes文件夹下。

关于安装：

任何代码的修改都需要在输入法英文状态下进行。

  1. 确保themes目录下有主题
  2. 打开博客根目录的_config.yml配置文件
  3. 将language: 后面改为zh-CN，将语言设置成中文。（冒号后面有空格，切记勿删）
  4. 将theme: 改为自己主题文件夹的名称。

里面的配置建议只修改Site下面几项，其余请勿修改，除非提到如何修改或自己懂得是什么意思。

  1. 在终端输入以下命令：
hexo cl && hexo g && hexo s

以上命令简称三连，每次对主题进行修改之后都需要执行一遍。

  1. 浏览器输入localhost:4000，进行本地预览主题

更多主题相关的设置则需要根据相应的主题文档进行，每个主题不一样，配置文件也不一样，后面就需要自己动手了。

部署云端

  1. 博客根目录右键点击Git Bash Here，输入
npm install hexo-deployer-git --save

安装部署插件，日后需要安装其它的插件也都需要在博客根目录执行相关命令，这样才会安装到正确的位置。

  1. 打开博客根目录的_config.yml，拉到底部，找到deploy: 配置项，进行如下配置
deploy:
  type: git
  repo:
    github: https://github.com/你的Github用户名/你的Github用户名.github.io.git,master

deploy前面不要有空格，所有:后面都要有空格。对齐缩进情况要严格按照示例来写，切记要在英文状态下输入标点，否则会出现报错。

虽然默认分支已改成main，但我仍然建议使用master

  1. 配置git用户名，任意位置右键，点击Git Bash Here，复制以下内容，把中文替换成自己的信息，按回车，输入正确不会有任何提示
git config --global user.name "用户名"
git config --global user.email "邮箱"
  1. 执行推送三连
hexo cl && hexo g && hexo d

三连的区别主要是最后要预览还是要推送来决定是s还是d

  1. 等待终端提示完成，在这过程中可能会需要你输入密码，注意提示，看不懂就用翻译！反正都是英语！按提示输入就好了！

这里也有可能会报spawn failed、time out、10054这三个类型的错，跟前面一样，多试几次就好（我当初也老会出现这个，都是一直试一直试，然后突然就成功的，要问我什么原因…我也不知道！）

  1. 不出意外的话到这里就部署完成了，浏览器输入用户名.github.io，即可看到你的网站。

如何写作

  1. 首先是要新建一个xxx.md的文件，有两种方法：
    1. 常规方法，新建文件将后缀改成.md
    2. 使用以下命令快速新建
hexo new post xxx

md是Markdown的后缀名，是一种简洁方便的文本标记语言。你只需要记住简单的几种语法，就可以快速进行编写。

为什么使用 Markdown？

首先 Hexo 本身就是将 Markdown 转化为静态的 Html 文件，来方便用户编写文章。

Markdown 就好似介于 Word 与 TXT 文本之间。Word 体积大、大部分功能实际上根本用不到，且最后
的文档常常带有许多冗余信息。

而 TXT 却无法实现加粗、标题、下划线、水平分割线等常用的功能。

综上所述！反正得用！常用的编辑器有前文提到的vscode和Typora，再贴两篇md语法，自用markdown手册、Markdown 教程。

  1. 打开.md文件，写上文件头，譬如标题、日期、分类等，具体内容可看官方文档，以下是本文的文件头示例：

    ---
title: Hexo+Github搭建个人网站
date: 2021-6-8
tags:
  - Hexo
  - 网站搭建
categories:
  - 网站相关
  - 搭建教程
---

  2. 将写好的文章放到博客根目录→source→_posts文件夹里。若是用命令新建的md文件会直接存放在该目录下。

  3. 在终端执行三连即可。

到这里整个流程就结束了，你需要做的就是存好你这个文件夹里面的所有东西！

如何保存源码并使用Github Action实现自动部署

将来如果你的文章越来越多，那么相应三连的时间也会越来越长。

解决这一难题的关键在于Github Aciton。简单来说，将你的源码上传到Github私密仓库保存的同时，会自动在云端启动Action进行自动部署，也就是说在云端执行三连。

之后每次在本地完成修改之后只需要将源码推送到Github上，就可以同时完成保存源码和部署网站两个操作！

  1. 设置win10查看隐藏文件夹

  2. 将博客根目录和主题根目录的.git文件夹删除

    例如我使用的主题文件夹叫yun，那么主题根目录位置如下：

    博客根目录→themes→yun→主题根目录

  3. 获取Token

    1. 登陆Github，鼠标指向头像 ，点击Settings
    2. 找到Developer Settings，点击
    3. 找到Personal access tokens，点击
    4. 点击generate new token，Token名称随意，下面全部打上勾，点击创建
    5. 保存好Token，只会显示一次

  4. 创建仓库，名称随意，描述随意，将Private选上，点击创建

    今后的源码将会上传到该仓库，因为创建了高权限的Token，所以该仓库务必选择Private

  5. 打开博客根目录的_conflg.yml，找到depoly配置项，设置成如下形式：

    # Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
  type: git
  repo: https://你的Github用户名:生成的Token@github.com/你的Github用户名/你的Github用户名.github.io.git,master

    虽然默认分支已改成main，但前文使用的是master，所以这里也使用master，务必前后一致。可直接将以上内容复制进去替换。

  6. 博客根目录新建.github文件夹，开头有个.请勿忽略

  7. 在.github文件夹内新建workflows文件夹

  8. 在workflows文件夹内新建autodeploy.yml文件

  9. 将以下内容复制到autodeploy.yml，注意替换中文标注的信息

    # 当有改动推送到master分支时，启动Action
name: 自动部署

on:
  push:
    branches:
      - master

  release:
    types:
      - published

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - name: 检查分支
      uses: actions/checkout@v2
      with:
        ref: master

    - name: 安装 Node
      uses: actions/setup-node@v1
      with:
        node-version: "12.x"

    - name: 安装 Hexo
      run: |
        export TZ='Asia/Shanghai'
        npm install hexo-cli -g

    - name: 缓存 Hexo
      uses: actions/cache@v1
      id: cache
      with:
        path: node_modules
        key: ${{runner.OS}}-${{hashFiles('**/package-lock.json')}}

    - name: 安装依赖
      if: steps.cache.outputs.cache-hit != 'true'
      run: |
        npm install --save

    - name: 生成静态文件
      run: |
        hexo clean
        hexo generate

    - name: 部署
      run: |
        git config --global user.name "你的Github用户名"
        git config --global user.email "你的Github邮箱"
        git clone https://github.com/你的Github用户名/你的Github用户名.github.io.git .deploy_git
        # 此处务必用HTTPS链接。SSH链接可能有权限报错的隐患
        # =====注意.deploy_git前面有个空格=====
        # 这行指令的目的是clone博客静态文件仓库，防止Hexo推送时覆盖整个静态文件仓库，而是只推送有更改的文件
        hexo deploy

  10. 添加屏蔽项

    1. 打开博客根目录的.gitignore
    2. 将以下内容复替换进去
      .DS_Store
Thumbs.db
db.json
*.log
node_modules/
public/
.deploy*/
.deploy_git*/
.idea

  11. 将源码推送到私有仓库，操作如下

    1. 博客根目录鼠标右键，打开Git Bash Here
    2. 输入git init按回车（初始化git，会在博客根目录创建隐藏的.git文件夹）
    3. 输入git remote add origin https://github.com/你的Github用户名/你的私密仓库名字.git按回车（建立远程连接。）
    4. 输入git checkout -b master按回车（切换master分支）
    5. 输入git add .按回车（添加缓存，注意后面有空格和.）
    6. 输入git commit -m "这次做了什么更改，简单描述下即可"按回车
    7. 输入git push --set-upstream origin master按回车（推送至私密仓库）
    8. 大功告成！

  12. 之后每次推送都要后三条命令，难免会觉得有些麻烦，现在创建一个脚本，操作如下

    1. 博客根目录新建update.sh
    2. 复制以下内容粘贴进去
      # 如果没有消息后缀，默认提交信息为 `:pencil: update content`
info=$1
if ["$info" = ""];
then info=":pencil: update content"
fi
git add -A
git commit -m "$info"
git push origin master
    3. 之后使用的话，博客根目录启动终端，输入sh datesh.sh即可。

换电脑后如何继续使用

换电脑之后怎么继续发博客，这是本章节要解决的内容。

如果你是小白，并且对Github并不熟悉，我相信你会需要这样的内容，因为这也是我曾经遇到的问题。

目前我会的方法只有两个：

执行以下任意方法前的必要条件

  1. 安装Node.js
  2. 安装Git
  3. 终端执行npm install hexo-cli -g
  4. 终端执行git config --global user.email "你的Github邮箱"
  5. 终端执行git config --global user.name "你的Github用户名"

无论何时何地（手机除外），想要使用 Hexo 发文章或进行修改，都需要安装有以上两个软件，并在安装好软件后执行后三条命令。

以下方法均基于具备上述条件的情况下使用。

方法一：（针对没有进行自动部署的操作）

  • 直接将整个文件夹压缩成压缩包，在适当的环境中解压出来使用。（就这么简单！）

方法二：（适用于按照本文完成自动部署操作后）

  1. 任意位置新建文件夹，并在文件夹内启动终端，将以下命令替换成自己的信息，输入并按回车

    git clone https://你的Github用户名:你的密码@github.com/你的Github用户名/你的私密仓库名称.git

    因为这是私密仓库，所以下载的命令稍微有点不一样，需要密码

    Github上将文件下载下来简称 clone 或中文叫克隆。

  2. 等待下载完毕，中途有可能会因为网络原因报错，多尝试几次即可

  3. 下载完毕你会看到出现了一个文件夹，点进去再启动终端

  4. 输入npm install回车

    这一步实际上是将Hexo的插件重新安装回去，因为在DIY别人主题的时候难免会安装不同插件，需要重新装回方可正常运行。

    博客根目录的node_modules是插件目录，因为我们在推送私密仓库时会将其屏蔽，所以一开始clone下来不会有node_modules目录，也就是没有插件。

    package.json则是记录安装过什么插件，在npm install时会将所有记录的插件重新安装，请勿修改该文件内容。

  5. 安装完插件执行一下预览三连，本地能正常访问则没有太大问题，后续需要推送源码或部署网站只需在博客根目录启动终端并输入sh update.sh执行前文的脚本即可。

如何绑定域名

域名=网址，域名是要钱的！但也不是很贵，具体可以到阿里云、腾讯云上找找看，使用需要备案，备案流程自行百度。也有不需要备案的提供商，但我不认识，自行百度！

可能会有人觉得xxx.github.io并不是很好看，或者是想要拥有一个能彰显自己的域名，一打开它就能看到网站，那么你需要以下操作。

科普：什么是一级域名，譬如本站imlili.top，imlili是域名主体，.top是后缀，那么合起来就是一级域名。

什么是二级域名，譬如本站域名是www.imlili.top ，前面多加www，也就是往前多加一级，就是二级域名，以此类推。所以本站用的是二级域名而非一级域名。

以下操作以腾讯云为例

  1. 在博客根目录的source文件夹内新建文件CNAME（没有任何后缀）
  2. 用记事本打开CNAME文件，填入你的域名，保存
  3. 打开腾讯云域名解析，找到你的域名，点进去
  4. 点击添加纪录,分以下两种情况
    1. 如果是使用一级域名，则
      1. 主机记录填@
      2. 记录类型选CNAME
      3. 记录值填你的Github用户名.github.io
    2. 如果是使用二级域名，则
      1. 主机记录填www（或其它任何你想要的前缀）
      2. 记录类型选CNAME
      3. 记录值填你的Github用户名.github.io
  5. 保存添加纪录，执行三连重新部署一次网站
  6. 打开Github，打开部署静态网页的仓库（公开的）
  7. 点击Settings，再点击Pages
  8. 在Custom domain处填入自己的域名
  9. 稍等片刻就可以用自己的域名访问网站了
