无需服务器部署快速部署高仿网易云音乐

2021-06-20
1.6k- 5m-
网站相关 > 搭建高仿网易云-高仿网易云Vue

前言

  在网上看到别人的高仿网易云觉得挺有意思的，想着自己也搞一个，随即打开了Github进行一番搜寻，搭建过程较为简单，中间也出现了bug，随即决定写下这篇文章记录一下。

  文章分为自建API和正式搭建两部分，本来是想着用的别人的API省事儿的，结果昨天才搭建完，今天API就失效了！索性自己建一个！

效果

  具体效果前往本站实验室查看：传送门

参考项目：

  • NeteaseCloudMusicApi：传送门（此项目为API项目）

  • vue-netease-music：传送门（此项目为网易云音乐高仿项目）

必备前提

  1. 下载安装 Node.js，建议下载长期支持版（LTS），安装全程点下一步就完事儿了！

  2. 下载安装 Git，安装就一直点下一步！（如果官网下载比较慢，可以试试这里）

  3. Github账号，如没有，自行注册

  4. Vercel账号，如没有，自行注册

    如何判断是否安装成功以上软件，打开终端：

    1. 输入node --version，提示版本号则Node.js安装成功

    2. 输入git version，提示版本号则Git安装成功。

  5. 配置git用户名，任意位置右键，点击Git Bash Here

    1. 输入git config --global user.name "用户名"按回车执行
    2. 输入git config --global user.email "邮箱"按回车执行

自建API

想了解什么是API可自行百度，高仿网易云没有API将无法正常运行

具体步骤：

  1. 确保已登陆Github，打开NeteaseCloudMusicApi，点击右上角的Fork，稍等片刻等待它变成你的仓库
  2. 打开Vercel，用你的Github账号登陆Vercel，中间可能需要关联一下信息，根据提示操作即可
  3. 在Vercel官网点击New Project
  4. 接着左边会有Import Git Repository，选择你刚才Fork的项目，并点击import
  5. 点击PERSONAL ACCOUNT的select
  6. 接着再直接点击Continue
  7. PROJECT NAME随意填写
  8. FRAMEWORK PRESET选择Other,其余全部默认，然后直接点击Deploy，等待部署完成
  9. 部署完成后屏幕中间会有两个链接，复制第二个链接到浏览器打开，如出现这个服务已经成功跑起来了即成功
  10. 记住第二个链接，后面要用！！

正式搭建

本地搭建

  1. 桌面按下鼠标右键，点击Git Bash Here，复制或输入以下命令，按回车执行（克隆项目到本地）

    git clone https://github.com/sl1673495/vue-netease-music.git

  2. 打开克隆下来的vue-netease-music文件夹，在文件夹内启动cmd，复制或输入以下命令,按回车执行，等待一段时间后会多出node_modules文件夹（安装需要的依赖）

    npm i

  3. 接着继续在cmd内复制或输入以下命令，按回车执行（安装cnpm）

    npm install -g cnpm --registry=https://registry.npm.taobao.org

  4. 继续输入以下命令，按回车执行（安装node-sass模块）

    cnpm install node-sass

    第3.4步是安装node-sass模块，因为我发现不进行这两步会提示Error: Cannot find module 'node-sass'，导致后面无法build。

  5. 打开vue-netease-music文件夹

  6. 再打开utils文件夹

  7. 找到并打开axios.js文件

  8. 找到const BASE_URL = 并将后面的链接删除，填入前面获取到的链接，按Ctrl s保存（单引号请勿删除，=后面有个空格

  9. 在vue-netease-music文件夹打开cmd，并输入以下命令，按回车执行（打包文件）

    npm run build

  10. 得到music文件夹即表示成功

部署上线

具体步骤：

  1. 建立公开的Github仓库，名称随意，可参考这里

  2. 推送Github仓库，步骤如下

    1. 在前面生成的music文件夹内鼠标右键，打开Git Bash Here
    2. 输入git init按回车（初始化git，会创建隐藏的.git文件夹）
    3. 输入git add .按回车（添加缓存，注意后面有空格和.）
    4. 输入git commit -m "first commit"按回车
    5. 输入git branch -M main按回车（切换main分支）
    6. 输入git remote add origin https://github.com/你的Github用户名/你刚才建的仓库名.git按回车（建立远程连接。）
    7. 输入git push -u origin main按回车（推送至仓库）

      最后一步有可能会因为网络原因报错，解决方法点击这里，相关介绍在蓝色框内，道理是一样的

  3. 到这里你已经完成了远程推送，接下来是将其部署到Vercel，和前面自建API的步骤是完全一样的，从第三步开始，将项目选成你刚才远程推送的即可

  4. 后面只要打开能正常加载即表示成功！

后话

  整个搭建还是比较简单的，唯一的不可抗因素可能就是加载Github和Vercel比较吃力，因为毕竟是国外的，但是一般多刷新几次也都可以。

  说一下今天早上遇到的问题，昨天第一次就搭建成功，这对于作为一个才接触计算机不久的人来说还是比较高兴的。但今天早上再次打开的时候发现一直是转圈加载，开始还以为是我的网络慢了一点，直到后来过了好一会儿出现了error。

  一开始还以为我搭建有问题，然后看了Github项目里的dome也是同样问题，随即打开F12，好家伙！api红红的一片，没办法就去看了看NeteaseCloudMusicApi，想着把api给换了，现在回想起来好像挺简单的，但我第一次看的时候真的是看不懂！包括api放哪儿都不知道！

  随着内容的正常加载，心里还是有点小高兴的，因为某种程度上也算是第一次独立排查并解决了一个小小小bug（虽然只是换api的事）。所以写出了这篇文章记录一下，虽然语言啰嗦步骤细分也多，但我觉得这样应该会让刚接触的小白更好理解！因为到现在为止我也还是小白，还是希望能够在我的理解范围内让内容更具体更面面俱到一些😜🔚
